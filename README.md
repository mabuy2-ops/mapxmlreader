# 地籍図XML → DXF 変換アプリ

法務省地籍図XMLファイルをCAD用DXFファイルに変換するアプリケーションです。  
**Python不要・サーバー不要** — `index_standalone.html` をブラウザで開くだけで使えます。

---

## 起動方法

`index_standalone.html` をダブルクリックしてブラウザで開くだけです。

> 初回のみインターネット接続が必要です（JSZipライブラリをCDNから自動取得します）。

---

## 使い方

1. ブラウザでアプリを開く
2. ZIPファイル（または展開済みXMLファイル）をドロップゾーンにドラッグ＆ドロップ、またはクリックして選択
3. **「変換してDXFをダウンロード」** ボタンを押す
4. 変換完了後、自動的にDXFファイルがダウンロードされます

変換には概ね **20〜40秒程度** かかります（ファイルサイズ・PC性能による）。

## 仕様

### 入力XMLの構造（法務省地籍図XMLフォーマット）

```
<地図>
  <空間属性>
    <zmn:GM_Point>   … 座標点（公共座標系 X=北, Y=東）
    <zmn:GM_Curve>   … 線分（直接座標 or 点IDへの参照）
    <zmn:GM_Surface> … 面（曲線IDの参照リスト）
  </空間属性>
  <主題属性>
    <筆>       … 土地区画（大字名・地番・精度区分・面IDへの参照）
    <筆界線>   … 境界線（曲線IDへの参照・線種別）
    <筆界点>   … 境界点（点IDへの参照・点番名）
    <基準点>   … 基準点（点IDへの参照・名称・種別）
  </主題属性>
  <図郭>       … 図郭枠（地図番号・縮尺・四隅座標）
</地図>
```

### 座標変換

公共座標系では X が北方向、Y が東方向のため、DXF出力時に入れ替えを行います。

```
DXF_X = 公共座標_Y（東方向）× 1000  [mm]
DXF_Y = 公共座標_X（北方向）× 1000  [mm]
```

座標はJW-CAD互換のためメートルからミリメートルに変換し、データ左下が原点付近に来るようオフセットします。

### 出力DXFレイヤー構成

| レイヤー名 | 意味 | 色 | エンティティ |
|-----------|------|----|-------------|
| `BOUNDARY`   | 筆界線       | 赤（1） | LINE |
| `LANDNUMBER` | 地番テキスト | 緑（3） | TEXT |
| `BND_POINT`  | 筆界点       | 青（5） | POINT |
| `REF_POINT`  | 基準点・多角点 | 黄（2） | POINT / TEXT |
| `FRAME`      | 図郭枠・地図番号 | 白（7） | LINE / TEXT |

> レイヤー名はASCII英字を使用しています（JW-CADはUTF-8日本語レイヤー名を正しく認識できないため）。

### DXF形式

- フォーマット: **DXF R12（AC1009）** — JW-CAD・AutoCAD双方で確実に読み込める形式
- 線分: **LINE エンティティ**（LWPOLYLINEはJW-CADで表示されないため不使用）

### テキスト高さ

図郭の縮尺分母から自動算出します。

```
テキスト高さ = 縮尺分母 × 3  [mm]
```

例: 縮尺 1/500 → テキスト高さ 1500mm（印刷時 3mm に相当）

## 注意事項

- JW-CADで開く際、最初は全体表示（`Ctrl+F`）で確認してください。
- ブラウザのセキュリティ制限により、一部の古いブラウザでは動作しない場合があります。
- 変換処理中はブラウザが一時的に応答しなくなる場合があります。そのままお待ちください。
